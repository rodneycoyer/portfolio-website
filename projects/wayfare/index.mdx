Wayfare (travel) is a react application desigend for users to sign up for different experiences. Similar to a travel agency, users can:

* view resources
* view resource details
* login
* upload forms
* comment
* add favorites
* join mailing list

## Application Architecture

The Node/Express App and React App each has its own repository.

## Backend Development

### Node/Express <a href="https://nodejs.org/en/" target="_blank" rel="noreferrer"><img style="border-radius: 999px; vertical-align: -12px" src="https://w7.pngwing.com/pngs/780/57/png-transparent-node-js-javascript-database-mongodb-native-miscellaneous-text-trademark.png" alt="nodejs logo"/></a> <a href="https://expressjs.com/" target="_blank" rel="noreferrer"><img style="border-radius: 999px; vertical-align: -12px" src="https://w7.pngwing.com/pngs/925/447/png-transparent-express-js-node-js-javascript-mongodb-node-js-text-trademark-logo.png" alt="express.js logo"/></a>

User HTTP requests are made to a Node.js server. From here, RESTful API's respond back to the client with an appropriate response.

I used an SSL certificate to setup secure and encrypted links for HTTPS requests.

### Authentication <a href="https://www.passportjs.org/" target="_blank" rel="noreferrer"><img style="vertical-align: -12px" src="https://avatars.githubusercontent.com/u/1160530?s=280&v=4" alt="passport.js logo" width="40" height="auto"/></a> <a href="https://jwt.io/" target="_blank" rel="noreferrer"><img style="vertical-align: -12px" src="https://coursework.vschool.io/content/images/2016/06/jwt.png" alt="jwt logo" width="40" height="auto"/></a>

Passport middleware is used to securely create users, salt and hash passwords that are strored in the database. When a user is logged in, Passport authenticates the user using a local-strategy. If successful, a JWT is signed, generated and stored in the users browser during their session.
